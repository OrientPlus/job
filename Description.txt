================================================================
|			Краткое описание			|
================================================================

Заметка тип 1 - не запоролена, доступ имеет каждый
Заметка тип 2 - доступ только для создателя
Заметка тип 3 - доступ по специальному паролю. Доступ может получить любой, кто знает специальный пароль.

1. После утсановления соединения сервер выдает публичный ключ протокола RSA.
Он используется клиентом единажды для шифрования данных авторизации. Далее для шифрования передаваемых данных и клиентом и сервером используется симметричная криптография (AES). 

2. Каждое новое подключение клиента выделяется в отдельный поток из пула.

3. Оба контейнера с информацией прав доступа сохраняются в JSON файлах.

4. Создаваемые клиентами заметки по умолчанию хранятся в памяти сервера. При нажатии кнопки все заметки сохраняются в файлах.

5. Файлы хранящие данные сервера сохраняются в зашифрованном виде. Используется так же алгоритм AES, в качестве ключа - внутренний ключ сервера.
